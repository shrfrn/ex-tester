## Implementation Details

### Stack Overflow Error Detection
Check for:
- `error instanceof RangeError`
- `error.message.includes('Maximum call stack size exceeded')`

Provide message:
```
Stack Overflow Error: Maximum call stack size exceeded. This usually happens when a function calls itself infinitely (infinite recursion). Check if any function is calling itself without a proper stopping condition.
```

### Function Parameter Validation Algorithm
1. Extract function string with `context[functionName].toString()`
2. Match parameter list: `/function\s*[^(]*\(\s*([^)]*)\s*\)/` or `/\(\s*([^)]*)\s*\)\s*=>/`
3. Parse parameters: `paramList.split(',').map(p => p.trim())`
4. Check parameter usage: for each param, test if `new RegExp(`\\b${param}\\b`)` matches function body
5. Check for prompt: `/prompt\s*\(|confirm\s*\(/.test(functionStr)`
6. Apply logic:
   - If `expectedParamCount > 0` AND `hasPrompt` AND `!parametersUsed` → Error
   - If `expectedParamCount > 0` AND `hasPrompt` AND `parametersUsed` → Warning
   - Otherwise → Valid

### Early Exit Pattern
Apply to all tests that:
1. Call `runScript()` to execute student code
2. Check execution success with `checkAndRecord()`
3. Continue with additional tests that depend on successful execution

Do NOT apply to tests that:
- Only perform static code analysis
- Don't depend on execution context

## Testing Strategy
1. Test with student code containing infinite recursion (like the example in `Daniel Peretz/ex/16.js`)
2. Test with functions that incorrectly use prompt instead of parameters
3. Test with functions that correctly use both parameters and prompt
4. Verify early exit prevents cascading test failures
5. Verify error messages are clear and actionable

## Success Criteria
- Stack overflow errors display helpful message explaining infinite recursion
- Tests exit early when code execution fails, avoiding cascading failures
- Functions using prompt instead of parameters are detected and flagged with clear error
- Functions legitimately using both parameters and prompt are allowed with optional warning
